# [Bare-metal] 라즈베리 파이 4: 완벽한 침묵(Silent Boot)을 뚫어낸 UART 커널 트러블슈팅 일지

## Issue Description
라즈베리 파이 4(AArch64) 환경에서 C++ 기반의 베어메탈(Bare-metal) 커널을 직접 빌드하여 부팅을 시도했다. 1차 목표는 USB to UART 통신을 통해 맥북 터미널에 `System Initialized` 문자열을 출력하는 것. 

하지만 어떤 에러 로그도 없이 화면이 멈추는 **'완벽한 침묵(Silent Boot)'** 및 I/O 데드락 상태에 빠졌다. OS가 없는 순수 물리 메모리 환경이므로 커널 패닉 메시지조차 기대할 수 없는 상황. 하드웨어 물리 계층(L1)부터 기계어 부팅 시퀀스(L0)까지 전 구간을 강제로 해체하며 원인을 추적했다.

---

## Deep-dive Debugging & Tracing

### Phase 1: 하드웨어 결선(L1) 및 전력 점유 충돌 우회
* **현상:** 라즈베리 파이의 쿨링 팬이 이미 5V 핀과 6번 핀(GND)을 선점하여 물리적인 핀 할당 충돌이 발생.
* **해결 (Dynamic Rerouting):** 스로틀링 방어를 위한 쿨링 시스템을 유지하기 위해 우회로를 개척했다. UART 통신 파이프라인을 8번(TXD), 10번(RXD), 14번(새로운 GND) 핀으로 동적 재할당하여 물리 계층 충돌을 제거함. 추가로 모듈의 TX/RX를 쇼트시키는 루프백(Loopback) 테스트로 외부 장비의 결백을 증명했다.

### Phase 2: 컴파일 파이프라인 및 링커(L3) 핫픽스
* **현상:** 물리 결선 정상화 후에도 영원한 먹통. CPU가 예외(Exception)로 추락해 즉사한 것으로 추정.
* **원인 분석:** 가장 밑바닥 코드인 `boot.S` (스택 포인터 초기화)와 `linker.ld` (메모리 주소 할당)에서, 다중행 주석(`/*`)을 닫는 `*/` 단 2바이트가 누락되어 있었다.
* **결과적 붕괴:** 컴파일러와 링커가 주석 아래의 핵심 메모리 매핑 코드(`. = 0x80000;`)를 통째로 무시했다. 이로 인해 GPU가 커널을 `0x80000`에 로드했음에도, CPU는 엉뚱한 `0x0` 번지를 참조하며 하드웨어 폴트(Hardware Fault)에 빠진 것.
* **해결:** 주석 핫픽스 및 `linker.ld`의 `.bss` 섹션에 8바이트 정렬(`ALIGN(8)`) 패딩을 추가하여 0 초기화 루프의 오버슈트(Overshoot) 가능성을 원천 차단함.

### Phase 3: 펌웨어(L2)의 기만과 클럭(Clock) 동기화 추적
* **현상:** 메모리 누락을 고치고 ACT LED(초록불)가 정상 점등되어 커널이 살아남을 확인했으나, 터미널은 여전히 침묵.
* **원인 분석:** GPU 부트로더 2nd stage 로그(`uart_2ndstage=1`)를 강제 개방하여 펌웨어의 자백을 받아냈다. `config.txt`에 통신 속도를 `115200`으로 명시했음에도 브로드컴 펌웨어가 내부 클럭 분주비를 잘못 계산하여 **`103448.3 Hz`**라는 기괴한 주파수로 데이터를 쏘고 있었다. 수신 측(맥북)에서 주파수 불일치로 데이터를 전부 Drop 하고 있었던 것.

### Phase 4: 극한의 L0 샷건 타격과 하드웨어 하이재킹 (Root Cause)
* **현상:** C++ 코드 단에서 UART0 칩(PL011)의 레지스터(`0xFE201000`)를 강제로 제어하여 주파수를 115200으로 맞추려 시도했으나 실패.
* **원인 분석:** 추상화를 모두 걷어내고, 가장 무식한 투 트랙(Two-track) 물리 타격을 시도했다. 무한 루프 속에서 UART0(`0xFE201000`) 레지스터에는 'A'를, UART1(`0xFE215040`) 레지스터에는 'B'를 무자비하게 밀어 넣었다.
* **결과:** 터미널에 쏟아진 글자는 **'B'**. 즉, 펌웨어가 우리가 꽂아둔 14/15번 물리 핀을 고성능 칩인 UART0가 아니라, 보조 칩인 **UART1 (Mini UART)**에 하드와이어링(Muxing) 해두었다는 절대적인 팩트를 밝혀냈다.
* **최종 픽스:** 타겟 메모리 주소를 UART1(`0xFE215040`)의 레지스터 구조에 맞춰 C++ 폴링(Polling) 로직을 전면 재조립했다.

## 💡 Result & Insight
* **Status:** 재빌드 후 `System Initialized. Welcome to Pi OS.` 문자열이 1비트의 오차도 없이 출력되며, 침묵을 부수고 베어메탈 제어권(Root)을 완벽하게 확보함.
* **Insight:** 프레임워크가 제공하는 안락한 에러 로그 뒤에는 링커의 메모리 매핑, CPU/GPU의 부팅 시퀀스 차이, 하드웨어 핀 Muxing이라는 무자비한 로우 레벨의 진실이 숨어 있다. 에러 로그 한 줄 없는 심연에서, 추측이 아닌 물리 계층부터의 '논리적 격리(Isolation)'와 '레지스터 직접 타격'만이 유일한 해결책임을 증명한 잔혹한 디버깅이었다.
